What is SIDKick pico?

The SIDKick pico is a drop-in replacement for the SID 6581/8580 sound chips in Commodore 64 and 128 computers. It can also emulate the SFX Sound Expander (FM). Based on an extended version of reSID 0.16, it provides high-quality sound emulation at an affordable price.

Key Features

  • Dual SID Emulation: 6581 and/or 8580 based on extended reSID 0.16 (with parts from reSID 1.0)
  • Processor: RP2040 ARM Cortex-M0+ Dual Core, overclocked to 300 MHz
  • Memory: 2MB Flash + 264KB RAM
  • Advanced Filters: 19x measured 6581 filter curves (by Antti S. Lankila)
  • FM Synthesis: SFX Sound Expander emulation (6581/8580 + FM)
  • PRG Storage: Built-in launcher can store and run up to 16 PRG files
  • 2nd-SID Addresses: $d420, $d500, $de00, $df00, or custom via external chip select
  • Paddle/Mouse Support: Full analog input functionality with configurable filtering
  • Configuration Menu: SYS 54301 or SYS 54333 (works in both C64 and C128 modes)
  • Digi Boost: Optional extension for enhanced digi-playing techniques
  • Price: ~10€ DIY component cost, or $20-60 pre-assembled depending on variant
💡 PRG Launcher: You can upload PRG files in two ways: (1) Bundle PRGs into the .uf2 firmware file using the included tool, or (2) Upload PRGs from a disk drive using the built-in configuration menu. Great for storing games, demos, or utilities!

Why SIDKick pico?

🔧 Original SID Chips Are Dying

All extant SID chips were manufactured roughly 40 years ago, and more fail every year. Original chips are becoming:

  • Increasingly Rare: No new SID chips have been manufactured since the 1980s
  • Expensive: Working chips command premium prices (often $50-150+ each)
  • Unreliable: Even "working" 40-year-old chips may develop issues
  • Two Incompatible Versions: 6581 (older C64) vs 8580 (C64C) - different voltages, not interchangeable

SIDKick pico offers an excellent way to keep the sound of the SID alive! It emulates both chip versions, works in any C64/C128, and costs a fraction of an original chip - while adding modern features like dual-SID stereo and FM synthesis.

"While many Commodore purists dislike using replacement chips, the reality is that preservation through quality emulation is now essential to keep these machines functional." - Hackaday

Firmware Upload Procedure

For Interface Boards (with Pico):
  1. Remove Pico from C64/C128 (if installed)
  2. Press and HOLD the 'Boot' button on the Pico
  3. Connect Pico to PC via USB (while holding Boot)
  4. Pico appears as USB drive
  5. Release Boot button
  6. Copy .uf2 firmware file to the USB drive
  7. Pico automatically reboots with new firmware
For SKpico2040DAC:
  1. Short the two "boot" pins on SKpico2040DAC
  2. Connect 4-pin USB connector (GND, D+, D-, 5V) to PC
  3. Board appears as USB drive
  4. Copy .uf2 firmware file
  5. Board automatically reboots
Optional: Use the programming adapter PCB with pogo pins for easier connection. Adapter needs 6 pogo pins (0.68mm needle diameter, 0.9mm tip).
CRITICAL: DO NOT connect the Pico to USB while it is plugged into your C64/C128! Remove it first, or it's fine if soldered to SKpico-PCB.

Adding PRGs to Firmware

You can integrate PRG files into the firmware for quick launching:

Method 1: Using skpicopatch Tool
  1. Create a file named prg.lst with list of PRG files
  2. Run: skpicopatch [firmware.uf2]
  3. Tool creates SKpicoPRG.uf2 with integrated PRGs
  4. Flash this new firmware to your SKpico
Method 2: From Configuration Tool
  1. Launch configuration tool (SYS 54301)
  2. Press F7 to access PRG management
  3. Load PRGs from drive 8
  4. PRGs are flashed to SKpico memory
Launching Integrated PRGs:
  • SYS 54333,0 - Launch first PRG
  • SYS 54333,1 - Launch second PRG
  • SYS 54333,N - Launch Nth PRG
  • Press F7 in config menu for PRG selection screen

Troubleshooting

Configuration Tool Won't Start

Symptoms: SYS 54301 or SYS 54333 doesn't work reliably

Solutions:

  • Check all solder joints on the SKpico PCB
  • Verify correct orientation in SID socket
  • Check that Pico is firmly seated in socket
  • Try adjusting timing values using skpicopatch or SKpicotiming tool
  • Default timing values are 15 and 11 - try decreasing if problems persist

Mouse/Paddle Issues on Pico Clones

Symptoms: RGB LED doesn't blink, X-paddle affects Y-paddle, trigger mode doesn't work

Solutions:

  • RGB LED: Check RGB LED solder jumper is closed (often isn't on Chinese clones with black PCBs)
  • VREF Pad: Verify "VREF" solder pad has large resistor OR close with solder blob to enable trigger mode
  • Quality Control: Clone PCBs often have bad soldering, solder bridges, or missing components - inspect carefully
  • Paddle Diagnostics: Firmware v0.21+ allows paddle value offsetting via configuration tool
  • Diagnostic Harness: Use offset of ~20 for 100% pass rate on diagnostic tests (adjust as needed)
🔧 Pico Clone Compatibility: Many cheap Pico clones work fine, but quality varies. Some have wrong GPIO layouts (incompatible), missing components, or poor soldering. Always test with the "blink.uf2" example first. Genuine Raspberry Pi Pico or Pico 2 boards are recommended for best reliability.

Diagnostic ROM Reports "Bad SID"

Symptoms: Diagnostic ROM or harness shows out-of-range values

Cause: Old capacitors or aging components causing low readings

Solutions:

  • Check values in configuration tool diagnostic harness (should be 80-120)
  • Use skpicopatch to offset values statically or conditionally
  • Option 1: Static offset applied always
  • Option 2: Offset only when diagnostic situation detected

No Digi Sound in Old Tunes

Symptoms: Can't hear 4-bit samples in Ghostbusters or similar

Solution: Configure SKpico to use 6581 or 8580 with digiboost enabled

Glitches with Old SID Tunes

Symptoms: Very old SID tunes have audio glitches or don't play correctly

Cause: Old tunes reading write-only SID registers, timing issues

Solutions:

  • Adjust timing values using skpicopatch or SKpicotiming tool
  • Try smaller values than default (15, 11)
  • Some rare machines require different timing

Power Requirements

CRITICAL: The SIDKick pico is powered exclusively from the C64/C128 mainboard. DO NOT power from USB while installed in the computer!

It is safe to:

  • Connect Pico to USB when removed from C64/C128
  • Connect Pico to USB when Pico is soldered to SKpico PCB (but PCB not in C64/C128)
  • Flash firmware via USB when SKpico2040DAC is removed from computer

❓ Frequently Asked Questions

Which SIDKick pico variant should I buy?
Depends on your skill level and budget:
  • Never soldered: SKpico2040DAC pre-assembled ($40-60) - ready to use
  • Basic soldering: Interface v0.2 DAC + Pico ($15-20) - easy build
  • Want stereo audio: Any DAC variant (Interface v0.2 DAC or 2040DAC)
  • Budget build: Interface v0.2 PWM + Pico (~$10) - mono only
  • Latest tech: Wait for SKpico2350DAC (RP2350, coming soon)
Will this work in my C64/C128?
Yes! SIDKick pico works in all C64 and C128 models:
  • C64 "breadbin" (all revisions)
  • C64C (shortboard)
  • C128 (all models) - note: no $D500 address
  • C128D - may need to remove PSU standoff
  • Ultimate64 boards (requires firmware v0.223+)
Can I use a Raspberry Pi Pico clone?
Yes, but quality varies. Clones from AliExpress/Amazon work but may have issues:
  • Bad soldering or missing components
  • RGB LED requires different firmware (use _RGB_LED.uf2)
  • Test with Blink.uf2 before installing
  • Official Raspberry Pi Pico recommended for best reliability
How does it sound compared to a real SID?
Extremely close! SIDKick pico uses reSID 0.16 with reSID 1.0 components:
  • Blind tests: most people can't tell the difference
  • 19x measured 6581 filter curves (Antti S. Lankila data)
  • Purist review: "Most people will never notice any difference"
  • DAC variant offers better audio quality than PWM
  • See nIGHTFALL Blog review for technical analysis
What's the difference between 6581 and 8580?
Two incompatible SID versions:
  • 6581: Original (1982-1987), warm sound, requires 12V, distinctive filters
  • 8580: Later version (1987-1993), brighter sound, 9V only, different filters
  • SIDKick pico emulates both perfectly
  • Switch between them in config menu (SYS 54301)
  • Can even run dual 6581+8580 simultaneously!
Can I run dual SIDs at different addresses?
Yes! SIDKick pico supports multiple 2nd SID addresses:
  • $D400 (default, 1st SID)
  • $D420 (most common for dual SID)
  • $D500 (stereo SID standard, not on C128)
  • $D420 + $D500 simultaneously
  • $DE00, $DF00
  • Ultimate64: any address via special mode
  • Requires A5 wire for $D420, A8 for $D500
Do I need the DAC or is PWM good enough?
Depends on your ears and setup:
  • PWM (mono): Good quality, routes through TV/monitor, easy setup
  • DAC (stereo): Better quality, line-out option, required for true stereo
  • Most users prefer DAC for noticeably better audio
  • PWM is fine for casual use or budget builds
  • DAC adds ~$5 to cost but worth it

What's the difference between SIDKick pico and original SIDKick?
Two different products by the same creator:
  • SIDKick pico: Budget version (~$10-60), Raspberry Pi Pico, reSID 0.16+, dual SID + FM
  • SIDKick (Teensy): Premium version (~$80-120), Teensy 4.1, reSID 1.0, MIDI interface, LED strips, SAM speech

My paddles/mouse don't work correctly. What should I do?
Firmware v0.21+ includes paddle offsetting:
  • Enter config menu: SYS 54301
  • Adjust paddle offset values
  • Some setups need offset ~20 for 100% pass rate
  • Try VREF pad connection for trigger mode issues
  • Diagnostic harness can help identify correct offset
How do I update the firmware?
Easy process:
  • Download latest .uf2 from GitHub Releases
  • Remove SKpico from C64
  • Hold BOOTSEL button while connecting USB
  • Drag .uf2 file to Pico drive
  • Auto-reboots when complete
  • For SKpico2040DAC: use programming adapter or USB breakout

Filter Accuracy Deep Dive

🎛️ Why SID Filters Matter

The SID's filters are notoriously inconsistent. Every 6581 chip sounds slightly different due to manufacturing tolerances. This is part of the SID's charm!

The 19x Filter Curve Implementation

Technical Details (firmware v0.223):

  • Source Data: Antti S. Lankila measured 19 different physical 6581 chips
  • Measurement Method: Frequency sweep, -3dB points recorded for each filter mode
  • What's Captured:
    • Cutoff frequency variation (different per chip)
    • Resonance peaks (Q factor)
    • Filter mode differences (LP, BP, HP, notch)
    • Non-linear behavior at different settings
  • Implementation: SKpico interpolates between measured curves based on settings
  • Result: Each playthrough can sound like a different physical SID chip!

Technical Limitations & Trade-offs

What SIDKick Pico Does Perfectly:

  • ✅ Waveform generation (triangle, saw, pulse, noise)
  • ✅ ADSR envelope timing
  • ✅ Ring modulation
  • ✅ Sync waveforms
  • ✅ Voice mixing
  • ✅ Paddle/mouse potentiometers
  • ✅ Digi playback (with boost enabled)
  • ✅ 6581 vs 8580 differences

Minor Differences from Original:

  • ⚠️ Filter resonance peaks (very close, not identical)
  • ⚠️ Some subtle analog characteristics (leakage, crosstalk)
  • ⚠️ PWM version has slight noise floor (DAC eliminates this)
  • ⚠️ No true analog drift/warmth (debatable if this is a con!)

What It Can't Do:

  • ❌ Perfectly replicate specific physical chip's unique quirks
  • ❌ Age and drift like original (this is actually good!)
  • ❌ Satisfy extreme purists who want only original hardware

💡 The Practical Reality

For 99% of use cases - gaming, demos, music listening, casual composing - SIDKick pico with DAC is indistinguishable from much more expensive solutions.

The differences that exist are:

  • Extremely subtle
  • Only audible to trained ears on good equipment
  • Context-dependent (some songs show differences, most don't)
  • Often masked by the C64's own audio limitations

Value proposition: At $15-60, you get 95%+ of premium quality for 20-40% of premium price. That's extraordinary value.

🎹 Original SIDKick (Teensy 4.1) - Premium Version

💎 About the Original SIDKick

The original SIDKick is a premium, feature-rich SID replacement based on the Teensy 4.1. It was the first complete SID drop-in replacement that you can build yourself, created by the same developer (frntc) who later created the more affordable SIDKick pico.

GitHub: github.com/frntc/SIDKick

Key Features

🎵 Dual SID + FM Synthesis

Emulates two 6581/8580 SIDs simultaneously plus Sound Expander (Yamaha OPL-based FM) at the same time. Uses reSID 0.16 or reSID 1.0 (configurable).

🎹 MIDI Interface

Built-in MIDI interface emulation with breakout connector. Compatible with DATEL, Sequential, Namesoft interfaces (MC6850 UART, $DE0x).

💡 LED Strip Control

WS2812b LED strip support with multiple display modes including peak meter, color cycling, and more.

🔊 SAM Speech Synthesis

Optional SAM (Software Automatic Mouth) speech synthesis that announces profile selections.

⚡ 10 Profile Slots

Store and switch between 10 different configurations on-the-fly via optional button or menu system.

🎚️ Premium Audio

High-quality PT8211/TM8211 DAC with filters and operational amplifier. Available as stereo output or routed through mainboard (mono).

💡 PCM5102A Connection Tips

  • Power: 3.3V only! Do NOT use 5V (will damage chip)
  • XSMT Pin: Must connect to 3.3V or board stays muted - this catches many beginners!
  • FLT, DEMP, FMT: Usually tied to GND on cheap boards (normal/low latency mode)
  • SCK: Can leave unconnected (master clock - not needed)
  • Audio Output: ~1.6V line-level (good for amplifiers, not headphones directly)
  • Quality: Cheap boards ($3-5) often have "BB" marked chips (clones) but work fine for SIDKick

Step Component Tips & Techniques
1.1 Resistors & Capacitors (0603/0805) • Start with smallest first
• Apply tiny amount of solder to one pad
• Place component with tweezers

⚠️ Common SMD Mistakes to Avoid

  • Too much solder: Creates bridges between pins → Use less, add flux
  • Not enough heat: Cold joints (dull, grainy) → Increase temp to 350°C
  • Too much heat/time: Lifts pads, damages components → Quick touch, 2-3 seconds max
  • Wrong orientation: ICs have pin 1 markers → Always check before soldering!
  • Dirty tip: Solder won't flow → Clean tip on wet sponge frequently

Step 2: Pin Headers & Sockets (Top Side) - 20-30 minutes

From frntc: "The next step is to solder the pin header and sockets which works best if you follow these steps..."

  1. SID Socket Headers (14-pin, middle of PCB):
    • Insert headers from TOP side
    • Flip board over (rest on table)
    • Headers will be held in place by gravity
    • Solder ONE pin first, check alignment
    • If crooked, reheat and adjust
    • Solder remaining pins
    • Height critical: Flush to PCB for low-profile fit
  2. Pico Socket OR Direct Solder:
    • Socket Method (Removable Pico):
      • Use 2x 20-pin female headers
      • Insert from top, solder from bottom
      • Keep straight - check before soldering all pins
      • Allows firmware updates without removing from C64
    • Direct Solder (Lower Profile):
      • Solder Pico directly using castellated edges
      • IMPORTANT: Cut pin tips afterward
      • Add insulation tape over cut pins
      • Prevents shorts to C64 mainboard
      • Needed for flat C64c cases
  3. Additional Connectors (A5/A8/IO pads):
    • Solder wire directly to pads (easier)

Common Assembly Problems & Solutions

Solder bridges between SMD pins
Solution:
  • Add flux to the bridge
  • Use desoldering braid (solder wick)
  • OR heat both pins, let surface tension separate solder
  • Clean with IPA afterward
  • Prevention: Use less solder, more flux
SMD component "tombstoning" (standing up)
Cause: One pad heated before the other
Solution:
  • Heat both pads simultaneously if possible
  • OR solder one side, hold component while soldering other
  • If tombstoned: heat both pads, push component flat, let cool
Headers not straight/aligned
Prevention:
  • Solder ONE pin first
  • Check alignment before continuing
  • If crooked: reheat that pin, adjust, let cool
  • Then solder remaining pins
Lifted PCB pad
Cause: Too much heat/time, or pulling on component
Solution:
  • If trace still visible: solder wire from component to trace
  • If trace gone: find next connection point
  • Use 30 AWG wire for jumper
  • Prevention: Lower temp, shorter contact time
VREF pad issue on Pico clones (We Are All Geeks)
From community experience:
  • Cheap Pico clones may be missing VREF resistor
  • Causes paddle X/Y axis to interfere with each other
  • Check VREF solder pad on PCB
  • Should have resistor, OR close with solder blob for "trigger" mode
  • See troubleshooting section for details

How the SID Chip Works

SID Architecture Basics

SID = Sound Interface Device - the legendary MOS Technology 6581/8580 sound chip designed by Bob Yannes in 1981.

Component Function How It Works
3 Voices Sound generators Each voice is independent synthesizer
Can play different notes simultaneously
24-bit accumulator per voice
4 Waveforms Basic sound shapes Triangle: smooth, flute-like
Sawtooth: buzzy, brass-like
Pulse/Square: hollow, clarinet-like
Noise: white noise, percussion
ADSR Envelope Volume over time Attack: 0-8 seconds
Decay: 0-24 seconds
Sustain: 0-15 levels
Release: 0-24 seconds
Filter Tone shaping Low-pass, Band-pass, High-pass
12dB/octave slope
Resonance (feedback)
Cutoff frequency 30Hz-12kHz
Modulation Complex sounds Ring modulation (voice 1→2, 2→3)
Hard sync (resets oscillator phase)
Creates metallic, bell-like tones
Potentiometers Analog input 2 channels (X, Y)
8-bit resolution (0-255)
Used for paddles, mice

Register Map (Memory-Mapped I/O)

The SID is controlled by writing to memory addresses $D400-$D41C (54272-54300 decimal):

Address Register Purpose
$D400-$D406 Voice 1 Frequency, pulse width, waveform, ADSR
$D407-$D40D Voice 2 Same as Voice 1
$D40E-$D414 Voice 3 Same as Voice 1
$D415-$D416 Filter cutoff 11-bit frequency (low/high byte)
$D417 Filter routing Which voices go through filter
$D418 Volume + filter Master volume (0-15), resonance, mode
$D419-$D41A Paddles (read) Potentiometer X and Y values
$D41B-$D41C Voice 3 output Read oscillator/envelope (for visuals)

How reSID Emulation Works

What is reSID?

reSID is a software library that emulates the MOS 6581/8580 SID chip. Written by Dag Lem in the late 1990s, it's the gold standard for SID emulation.

Emulation Approach

  1. Cycle-Accurate Simulation:
    • Emulates SID at clock-cycle level (985,248 Hz PAL)
    • Each register write processed exactly as in real chip
    • Timing matches original hardware
  2. Waveform Generation:
    • 24-bit phase accumulator per voice (like original)
    • Frequency = increment value for accumulator
    • Waveform = lookup table based on accumulator value
    • Combined waveforms = bitwise AND/OR operations
  3. Envelope Generator:
    • Exponential curves (like analog original)
    • 15-step state machine per voice
    • Rate tables match chip measurements
  4. Filter Modeling:
    • reSID 0.16: Simplified filter model (faster)
    • reSID 1.0: Detailed transistor-level simulation (slower, more accurate)
    • SKpico: Extended 0.16 with 1.0 improvements + 19 measured curves

PWM Audio Explained

What is PWM (Pulse Width Modulation)?

PWM is a technique to create analog voltages using digital signals. The Pico rapidly switches a pin HIGH/LOW, and the average voltage represents the audio level.

How PWM Creates Audio

  1. Digital to "Analog":
    • Pico outputs 3.3V (HIGH) or 0V (LOW) very rapidly
    • Duty cycle = % of time spent HIGH
    • 50% duty cycle = 1.65V average
    • 25% duty cycle = 0.825V average
  2. PWM Frequency:
    • SKpico uses ~250 kHz PWM carrier frequency
    • Much higher than audible range (20 kHz max)
    • Low-pass filter removes carrier, leaves audio
  3. Filtering:
    • C64 mainboard audio circuit acts as filter
    • Removes 250 kHz PWM carrier
    • Passes 20 Hz - 15 kHz audio range
    • Result: analog-like audio signal

PWM Audio Quality Factors

Factor SKpico Implementation Impact on Sound
Resolution ~12-bit effective (after dithering) Good dynamic range, low quantization noise
Sample Rate ~44.1 kHz effective Captures full SID frequency range
Carrier Frequency 250 kHz Well above audible, easy to filter
Noise Floor ~-60 dB Slight hiss on quiet passages (most don't notice)

⚠️ Why PWM Isn't Perfect

  • Jitter: Slight timing variations in PWM edges
  • Filtering: C64 audio circuit not designed for PWM
  • Noise: Inherent to switching-based output
  • But: For $0 extra cost, sounds great to 90% of users!

I2S DAC Explained

What is I2S?

I2S = Inter-IC Sound - a digital audio protocol for transmitting audio between chips. Invented by Philips in the 1980s.

I2S Signal Lines

Signal Full Name Purpose
BCK Bit Clock Timing for each data bit (64x sample rate)
DIN (SD) Data In (Serial Data) Actual audio samples (16-bit per channel)
LCK (WS) Left/Right Clock (Word Select) Indicates left vs right channel (stereo)

How PCM5102A DAC Works

  1. Digital Input:
    • Receives 16-bit digital samples via I2S
    • 44.1 kHz sample rate (CD quality)
    • Separate left and right channels
  2. Digital-to-Analog Conversion:
    • 24-bit internal DAC (overkill for SID!)
    • Delta-sigma modulation
    • Very low distortion (<0.01%)
  3. Analog Output:
    • 1.6V RMS line-level output
    • DC-coupled (no output capacitors needed)
    • Differential outputs (L+, L-, R+, R-)
  4. Optional OpAmp Stage:
    • LM358 or TL072 amplifies signal
    • Converts differential to single-ended
    • Drives longer cables, headphones

Why DAC Sounds Better Than PWM

  • True Digital: No analog reconstruction from switching
  • Low Noise: 112 dB SNR (inaudible noise floor)
  • Flat Response: 10 Hz - 20 kHz ±0.1 dB
  • No Jitter: Stable crystal oscillator
  • Stereo: True independent left/right channels
  • Clean: <0.01% THD (vs ~1-2% PWM)

Bus Timing & C64 Integration

The Challenge: Phi2 Clock

The C64 expects SID to respond within specific timing windows:

  • Phi2 Clock: ~1 MHz (985,248 Hz PAL)
  • Setup Time: Data must be valid before clock edge
  • Hold Time: Data must stay valid after clock edge
  • Problem: Software emulation has variable latency!

SKpico's Solution

  1. GPIO Polling: Core 1 constantly monitors address/data bus
  2. Fast Response: <200 ns from write to processing
  3. Timing Modes: Firmware v0.223 has 3 modes (cycle with reset ~5s)
  4. Buffer: Small FIFO queue for register writes

💡 Why This Matters

Proper bus timing ensures:

  • No glitches in music playback
  • Compatibility with all C64 models
  • Works with accelerators (like SuperCPU)
  • Stable operation with Ultimate64

Firmware Naming Convention

📁 Understanding .uf2 Filenames

Format: SKpico[revision][output][LED].uf2

Examples:

  • SKpico15PWM.uf2 = Interface v0.15, PWM output, standard LED
  • SKpico15DAC.uf2 = Interface v0.15, DAC output, standard LED
  • SKpico15PWM_RGB_LED.uf2 = Same as first, but for RGB LED Pico clones
  • SKpico2040_15DAC.uf2 = SKpico2040DAC board, v0.15, DAC output

CRITICAL: Use the correct file for your hardware or it won't work!

How to Check Your Firmware Version

  1. Boot C64/C128
  2. Type: SYS 54301
  3. Config menu shows version at top/bottom
  4. Example: "SKpico v0.223"

Upgrade Procedure

For Removable Pico (Interface Boards):

  1. Power off C64
  2. Remove SKpico from SID socket
  3. Disconnect Pico from interface board
  4. Hold BOOTSEL button on Pico
  5. Connect Pico to PC via USB (while holding BOOTSEL)
  6. Pico appears as USB drive
  7. Download latest .uf2 from GitHub Releases
  8. Drag & drop .uf2 to Pico drive
  9. Pico auto-reboots with new firmware
  10. Disconnect USB, reconnect to interface board
  11. Reinstall in C64, test

For SKpico2040DAC (Integrated):

  1. Power off C64
  2. Remove SKpico2040DAC from SID socket
  3. Short "BOOT" pads with jumper/wire
  4. Connect USB (via programming adapter or direct solder)
  5. Appears as USB drive (keep BOOT shorted!)
  6. Drag & drop correct .uf2 file
  7. Auto-reboots
  8. Remove BOOT jumper, disconnect USB
  9. Reinstall in C64
