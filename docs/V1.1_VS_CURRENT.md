# SID Detector — V1.1 vs. Current (V1.4.26)

Comparison of `siddetectorv1_1.asm` (2385 lines, ACME) against `siddetector.asm`
(9720 lines, KickAssembler). Both build a C64 `.prg` that identifies SID chip
variants. See `CLAUDE.md` for the project overview and `STORY.md` for the
engineering narrative the current version draws on.

---

## 1. Summary

V1.1 (`siddetectorv1_1.asm`) is the archival reference. It is a single-file ACME
source, 2385 lines, with a detection chain that covered the chips that existed
circa 2020: SIDFX, ARMSID/ARM2SID, FPGASID, real 6581/8580, a noise-mirror
second-SID scan, a `$D418` decay fingerprint for emulators, and a partially
wired SwinSID Nano/Micro path. The reference release is
https://csdb.dk/release/?id=176909.

The current source (`siddetector.asm`) is on release **V1.4.26** (the
`v1.3.72` string at the top of the file is stale — the actual state is
V1.4.26, confirmed by the screen title at line 7990 and the `README.md`
header). It is ~4.1× the size of V1.1 after roughly 13 months of work,
converted to KickAssembler syntax, and covers **24+** chip/emulator variants.

Headline deltas:

- **New hardware detected:** SwinSID Nano (working), SIDKick-pico, BackSID,
  PD SID, KungFuSID, uSID64, ULTISID on Ultimate 64 (via UCI), plus
  CBM SFX Sound Expander / FM-YAM (Yamaha OPL at `$DF40/$DF50/$DF60`).
  ARM2SID SFX mode (SFX-only, SFX+SID) is decoded and displayed.
- **Detection-chain hardening:** carry-safe ordering, "real SID pre-check"
  at Step 0.25, SIDFX early-return guard, SIDFX-secondary probe chain
  (SID1=UNKN and SID2 slots get re-probed), multi-MixSID (D400+D420)
  bus-conflict handling, retry indicator on real-SID detection.
- **Stereo/multi-chip scanning** rewritten — ARM2SID's built-in config-map
  replaces naive scanning when available; fallback `fiktivloop` now types
  secondaries via full re-probe instead of guessing.
- **User-facing features:** sound test (T), per-chip info pages (I) with
  scrolling and navigation, readme/help screen (R), debug screens (D),
  colour-coded result rows, embedded TLR's original sid-detect2 on (L),
  embedded SID tune that plays under the UI, quit-to-BASIC (Q).
- **Toolchain migration** from ACME to KickAssembler (see §2).

## 2. Syntax / Toolchain Migration

Both files do the same job; directive syntax differs. The conversion is
mechanical and is noted at the top of `CLAUDE.md`. Concrete differences:

| Concept                       | V1.1 (ACME)                 | Current (KickAssembler)                   |
|-------------------------------|-----------------------------|-------------------------------------------|
| Comment                       | `;`                         | `//`                                      |
| Byte data                     | `!byte $09`                 | `.byte $09`                               |
| Word data                     | `!word $0801`               | `.word $0801`                             |
| Text (PETSCII/ASCII)          | `!text "FOO",0`             | `.text "FOO"` + trailing `.byte 0`        |
| Screen-code string            | `!scr "..."`                | `.encoding "screencode_upper"` + `.text`  |
| Symbol equate                 | `name = $A4`                | `.const name = $A4`                       |
| Origin                        | `* = $0801`                 | `* = $0801` (unchanged)                   |
| Label                         | `labelname`                 | `labelname:`                              |
| Char literal                  | `"A"`                       | `"A"` (same; also `#'A'` supported)       |
| Binary literal                | `%10000000`                 | `%10000000` (same)                        |
| Binary include                | —                           | `.var f = LoadBinary(...)` + `.fill ...`  |

The switch to KickAssembler enabled two features V1.1 could not have:
`LoadBinary` pulls in a `.sid` tune (`bin/Triangle_Intro.sid`) and TLR's
`sid-detect2.prg` as embedded binaries at assemble time (lines 37–46 of
`siddetector.asm`). V1.1 had no embedded music and no embedded alternate
detector.

Makefile notes: KickAssembler's jar is at `C:/debugger/kickasm/KickAss.jar`;
assembly is a single-command build. V1.1 would have used `acme`; the Makefile
in the current repo is KickAssembler-only. The original ACME source has been
preserved as `siddetector.asm.acme.bak` (referenced by CLAUDE.md) but is not
what the build uses.

All label declarations now end with a colon. Labels in V1.1 that were defined
without a colon (e.g. `start`, `basend`, `unknown`, `exit`, `end`, `armsidlo:`
is mixed both ways in V1.1) have been normalised to require `:` in the
current source.

Character strings in screen RAM changed from ACME's `!scr` directive (which
translates PETSCII→screencode at assemble time) to KickAssembler's
`.encoding "screencode_upper"` scoped block — see current lines 7989–8015
for the main screen definition.

## 3. Detection Chain — What's New

The V1.1 chain is short and strictly linear: `DETECTSIDFX` → `Checkarmsid`
→ `checkfpgasid` → `checkrealsid` → `checksecondsid` → decay fingerprint
(`calcandloop`). Everything from `ibegin:` (v1_1:158) to `funny_print:`
(v1_1:365) is the full chain.

The current chain has the same backbone but with guards, pre-checks, retry
loops, and several new probes in front of and behind each existing step.

### `DETECTSIDFX` — SIDFX SCI handshake
- **V1.1 (v1_1:881):** SCI PNP login is done once, produces `data1 = $30`
  or `$31`, captures `PNP+0..+3`, and optionally reads `$D41D/$D41E` for
  SID model fields, but **does not populate `sid_list`** from that
  information. The SIDFX secondary slot is not typed.
- **Current (`siddetector.asm:4553`):** Same SCI handshake, but on success
  `sidfx_d41d` and `sidfx_d41e` are saved, and a new function
  `sidfx_populate_sid_list` (`:6461`) is called later. That function
  translates SW1 bits into secondary address (D500/D420/DE00 per
  `sidfx_sec_hi`/`sidfx_sec_lo`) and runs a probe chain against that slot
  (`sfx_probe_skpico`, `sfx_probe_dis_echo`, `checkbacksid`, inline `$A5`
  KungFuSID test, `checkfpgasid`) to identify what kind of chip is inside
  the SIDFX cartridge. This work landed over V1.3.58–V1.3.70 (see
  STORY.md §19). V1.1 had no secondary-chip identification for SIDFX.

### `Checkarmsid` — ARMSID / ARM2SID / SwinSID Ultimate
- Both versions use the same "DIS echo" trick: write `D`/`I`/`S` to
  D41F/D41E/D41D, wait `loop1sek`×2 (~1.8 ms each) and read D41B back
  for `'S'` = $53 (SwinSID U) or `'N'` = $4E (ARMSID/ARM2SID), then
  D41C (`'W'`/`'O'`) and D41D (`'R'` for ARM2SID).
- **V1.1 (v1_1:464):** Pure DIS echo. No firmware-version readback.
  Self-modifies ~28 `cas_*` address bytes (v1_1:465–540). Result in
  data1/data2/data3, no further metadata.
- **Current (`:3778`):** Same DIS echo core, same ~28-site self-mod.
  Adds `armsid_get_version` (`:6193`) which enters config mode via
  `d`/`i`/`s` (lowercase — different from the detection trigger),
  polls for `'N'`/`'O'` ACK, issues further 2-char commands (`ei`/`ii`/
  `fi`/`gi`/`mm`/`lm`/`hm`) and stashes results into
  `armsid_major/minor/cfgtest/no_c/ei_b/ei_c/ii_b/ii_c/sid_type_h/
  auto_sid/emul_mode/map_l/map_l2/map_h/map_h2` (see data block
  `:7925–7940`). The `mm` readback gives ARM2SID's SFX capability mode;
  the `lm`/`hm` readbacks give a full nibble-packed map of 8 ARM2SID
  slot address/type entries, used by `arm2sid_populate_sid_list`
  (`:6342`) to build the stereo SID list directly from the chip rather
  than scanning for mirrors. This is the single largest detection
  addition in the current source.
- Also new: the `step2_armsid` block (`:361`) is preceded by MixSID
  workaround logic that enters "CS2-DIS" mode (writes D43F/D43E/D43D —
  the D420 mirror) to detect a real 6581 at CS1 while an ARMSID is at
  CS2. This is pure V1.3.74 territory — the SID1-snooping problem did
  not exist in V1.1's test matrix.

### `checkfpgasid` — FPGASID magic cookie
- **V1.1 (v1_1:605):** Writes `$81`/`$65` to D419/D41A, bit 7 to D41E,
  reads back `$1D`/`$F5` at D419/D41A, and D41F for `$3F` (8580) or
  `$00` (6581). Returns `$06`/`$07`/`$F0`. Self-mods 9 address sites.
- **Current (`:4244`):** Same magic cookie and same readback logic.
  Stores CPLD revision (`fpgasid_cpld_rev`) and FPGA revision
  (`fpgasid_fpga_rev`) and SID2 type (`fpgasid_sid2_type`) into ZP
  buffers for the debug page. Also handles being called from
  `sidfx_populate_sid_list` with `sptr_zp` pointing at a secondary
  slot (V1.3.66). The core register sequence is unchanged.

### `checkrealsid` — real 6581 / 8580
- **V1.1 (v1_1:691):** Writes `$48` (gate+sawtooth bit set) to D412,
  shifts right to `$24`, reads D41B with `and #$FE` threshold gate,
  checks `$03`/`$05`+`$06`/`$08` sequence for real-SID confirmation.
  Single attempt, no retry. Self-mods 9 sites.
- **Current (`:4340`):** Same sequence, wrapped in a **3-try retry
  loop** (`crs_retry:` at `:4393`). When a try fails because a VIC
  bad-line stole cycles during the D41B readback, it retries up to 3
  times and sets `retry_zp` to the number of attempts needed. The
  main screen appends a `*` after "6581 FOUND" / "8580 FOUND" if any
  retries were required (V1.3.83 — see TODO.md). Also called *early*
  (Step 0.25, main:300) as a pre-check to skip `checkswinsidnano`
  when a real SID is already confirmed, preventing false
  "SwinSID Nano found" on real 6581/8580 hardware. Also called
  inline inside the CS2-DIS window at step 2 (main:420) to handle
  MixSID configs.

### `checksecondsid` — noise-mirror second SID probe
- **V1.1 (v1_1:792):** Writes noise waveform + freq-hi, reads D41B
  mirror at secondary slot, counts up to 10 tries for a non-zero
  value. Self-mods 8 sites. Finds a "secondsid" (type `$10`) at any
  address but cannot say what kind it is.
- **Current (`:4464`):** Core logic unchanged. But in the chain it's
  now only the fallback for real-SID stereo. `sidstereostart`
  (`:4848`) now has an **early-return guard when SIDFX is primary**
  (V1.3.59 — fixes PDsid mirror overflow, see TODO.md "Multiple SID
  / stereo detection errors"); ARM2SID uses its config map;
  SwinSID Ultimate skips fiktivloop entirely (`end_skip_fiktiv:`
  in the primary chain) because the AVR OSC3 returns 0 with noise
  enabled and triggers false positives at D500.

### `calcandloop` / `ArithmeticMean` — D418 decay fingerprint
- **V1.1 (v1_1:1913):** Samples `$0400`/`$0401`/`$0402` three bytes
  incremented by tight loop while D418 decays from $1F. 6 repeats,
  16-bit mean via `ArithmeticMean`. Emulator dispatch table in
  `checktypeandprint` (v1_1:1980) covers VICE ResID/FastSID 6581/8580,
  HOXS64, Swinsid Ultimate, FPGASID, ULTISID range, C64DBG
  ResID/FastSID/ResIDfp.
- **Current (`:7515`):** Identical algorithm. The emulator dispatch
  table (`checktypeandprint:` at `:7593`) is essentially the same —
  nc_Swinsidn, nc_ULTIsid, nc_hoxs, nc_Residfp6581d, nc_Fast6581d,
  nc_Resid6581d, nc_Swinsidu, nc_FPGAsid, nc_Resid8580, nc_Resid6581,
  nc_FastSid. What changed is that this path is now **a fallback
  only**: by the time the decay fingerprint runs, all modern
  hardware has already been identified by probe. A new spinner
  (`decay_spinner` at `:8281` — 8-frame `*+/-*+/-`) displays progress
  during the measurement.

### `checkkungfusid` (new)
- **V1.1:** Not present.
- **Current (`:4121`):** Writes `$A5` (firmware-update magic) to D41D,
  waits ~6 ms via `rp_delay`, reads D41D back. `$5A` = new firmware
  byte-swap ACK; `$A5` = old firmware RAM-array echo. Either matches
  → `data1 = $0C` = KungFuSID. See STORY.md §11.

### `checkswinsidnano` (rewritten)
- **V1.1:** `checkswinmicro` (v1_1:1861) existed but was effectively
  disabled — invoked via `swinmicro:` at v1_1:319, which immediately
  jumps to `nosound` before the JSR. The V1.1 TODO even says
  "check SwinsidNano by eliminating others".
- **Current (`:4145`):** Real SwinSID Nano detection. 2-stage,
  3-retry algorithm using D41B OSC3 change-count at `$FFFF` noise,
  with a 50 ms wait between stage 1 and stage 2 to discriminate
  SwinSID Nano's continuously-running AVR oscillator from a settling
  NOSID bus. Gated by the Step 0.25 real-SID pre-check so real 6581
  LFSR speed never reaches this routine. Returns `data1 = $08`. See
  STORY.md §12 for the full discussion including the U2+ bus-noise
  false-positive edge case.

### `checkusid64` (new)
- **V1.1:** Not present. The V1.1 top-of-file TODO does not mention it.
- **Current (`:4087`):** Writes the 5-byte unlock sequence
  `$F0 $10 $63 $00 $FF` to D41F, waits ~3 ms, does two reads with a
  second delay, requires both in `$E0-$FC` and `|read2-read1| ≤ $02`.
  Returns `data1 = $0D`. Also called defensively inside the Step 0.5
  SwinSID Nano path (`csn_not_usid64:` at `:333`) because uSID64 can
  trigger the same OSC3 activity pattern as SwinSID Nano.

### `checkpdsid` (new)
- **V1.1:** Not present. The V1.1 chain has nothing between
  `checkfpgasid` and `checkrealsid`.
- **Current (`:3922`):** Writes `'P'` ($50) to D41D, `'D'` ($44) to
  D41E, reads back D41E for `'S'` ($53). Returns `data1 = $09`.
  Called from the main chain (at the `checkpdsid_step:` dispatch
  point, `:562`) and from the SIDFX SID1=UNKN probe chain.

### `checkbacksid` (new)
- **V1.1:** Not present.
- **Current (`:9436`):** Implements a reverse-engineered unlock/poll
  sequence: write `$02` to D41B, `$01` to D41C, `$B5` to D41D, `$1D`
  to D41E, re-arm D41B on each poll, read D41F up to 15×42 ms looking
  for the `$01` echo. Pre-check on cold D41F for the `$01` bus-float
  case (otherwise every NOSID bus would false-positive). Uses
  `(sptr_zp),y` indirect-indexed addressing for D41B writes so the
  same routine works at any SID slot address — used this way from
  `sidfx_populate_sid_list` to probe SIDFX secondary slots. Returns
  `data1 = $0A`. See STORY.md §9 for the disassembled-protocol
  history.

### `checkskpico` (new)
- **V1.1:** Not present.
- **Current (`:3984`):** Writes `$FF` to D41F to enter SIDKick-pico
  config mode, uses D41E as a **manual pointer** (write `$E0+i` to
  select byte `i` of VERSION_STR), reads D41D for data. Checks for
  `'S'` at offset 0 and `'K'` at offset 1. If confirmed, also
  performs a phase-3 read of `config[8]` to detect if FM Sound
  Expander mode is active. Returns `data1 = $0B` (8580 mode) or
  `$0E` (6581 mode when running under SIDFX). See STORY.md §10.

### `checkfmyam` — OPL detection (new)
- **V1.1:** The V1.1 top-of-file TODO lists "SFX Sound expander -
  half done" and "x-soundfx.asm" as an intended module. Not
  implemented.
- **Current (`:6725`):** Writes OPL register timer config through
  `cfm_write_reg`, masks Timer 1 IRQ (to avoid the OPL `/IRQ`-storm
  hang on real hardware), waits, reads `$DF60` for a status whose
  upper bits are clear. Two-read stability with `(status & $E0) == 0`
  is the acceptance criterion (STORY.md §23 has the full story of
  why this heuristic, after `== $00` exact match, `$C0`, and `$40`
  bitmask all had edge cases). Covers both CBM Sound Expander
  (YM3526) and FM-YAM (YM3812) with one label — they are
  indistinguishable at the register interface. Result stored in
  `fmyam_detected` (`:7952`).

### Addresses / guards / U64 detection (new top-of-start code)
The current main routine (`:246–335`) adds several new steps before
the "Step 1" SIDFX call:

- **Step 0.1 (Ultimate64 detection) `:273–286`:** Read `$DF00`, then
  `$DF1F` (UCI status register). On U64 these respond `$00`/non-`$FF`;
  on a bare C64 they're open bus (`$FF`). Sets `is_u64 = 1` if
  confirmed. Prevents VICE+SFX false positives (SFX maps all
  `$DF00-$DFFF` to OPL2, making `$DF1F` look like U64).
- **Step 0.25 (real-SID pre-check) `:289–307`:** Discussed above.
- **Step 0.5 (SwinSID Nano) `:310–335`:** Runs before any D41F writes
  because DETECTSIDFX and Checkarmsid both corrupt SwinSID Nano mode
  by touching D41F. Has a uSID64 veto in the middle.

V1.1 had no U64 awareness and no probe-ordering guards.

ULTISID detection in V1.1 was by the decay-fingerprint dispatch only
(ranges `$DA-$F1` in `nc_ULTIsid`). The current source keeps that
fingerprint as fallback but adds a proper UCI-based probe
(`check_uci_ultisid` at `:8034`) that queries the U64's
`CTRL_CMD_GET_HWINFO` and reads back the actual EMUSID1/EMUSID2
address+type from the hardware via `$DF1C`/`$DF1D`/`$DF1E`.

ARM2SID variants and firmware versions are captured but not dispatched
as separate `sid_list_t` codes — instead the ARM2SID reports its major/
minor version and emul_mode on the info page.

## 5. New Features (non-detection)

Things V1.1 did not have at all:

**Sound test (T key).** `sound_test_entry:` at `:3050` plays a 7-note
C-major arpeggio across 3 octaves on each detected SID (via
`snd_patch_page` self-modifying all 31 `sta $D4xx` in `st_soundtest`
to any SID page). Three waveform voices: pulse+ring, triangle,
sawtooth+ring. A matching FM sound test (`sfx_soundtest:` at `:3359`)
plays the same arpeggio pattern through the Sound Expander / FM-YAM
when detected, with three FM instruments (Flute/Organ/Bell). V1.1
had no audio output at all.

**Per-chip info pages (I key).** `info_entry:` at `:1304` renders up
to 19 pages (ip_nosid, ip_6581, ip_8580, ip_armsid, ip_swinu,
ip_swinano, ip_fpga8580, ip_fpga6581, ip_sidfx, ip_ulti, ip_vice,
ip_hoxs, ip_sidkpic, ip_pubdom, ip_backsid, ip_kungfusid,
ip_unknown, ip_usid64, ip_fmyam — table at `:8305`). Each page
includes chip name, revision quirks, firmware/update info,
"where to buy" links. Navigation is via B/M (prev/next — cursor keys
fall back because VICE gtk3 doesn't reliably map CRSR LEFT/RIGHT in
the CIA matrix). Header/footer are re-rendered on page flip
(V1.4.25). Scrolling stops before trailing blanks (V1.4.24).

**Readme / help screen (R key).** `readme_entry:` at `:2735` renders
a ~74-line scrolling text (`readme_text:` at `:9132`) that
summarises the detection chain and chip list. Header/footer,
scroll indicator, 23-line unbreakable V1.4.xx block.

**Debug screen (D key).** Two pages, `debug_entry:` at `:1842` and
`debug_entry_p2:` at `:2666`. Shows raw hex readings of D418,
POT X/Y, D41B/C/D/E/F, `data1/2/3/4`, `ArrayPtr1/2/3`, ArithMean,
emulator fingerprint, SIDFX D41D/D41E, ARMSID ei/ii results,
FPGASID CPLD/FPGA revisions, full `sid_list` dump, UCI response
for U64. Version string on page 1 (V1.3.78).

**TLR's original sid-detect2 (L key).** `tlr_entry:` at `:1201`
copies a second detector (loaded as binary at assemble time,
`:44-46`) to `$0801` and chains into it. V1.1 had no such
interop.

**Quit (Q key).** `do_quit:` at `:1168` restores blue border/screen
and jumps back to BASIC. V1.1 relies on a hard reset.

**Colour-coded results.** `colorize_rows:` at `:3506` reads column 13
of each result row and sets `$D800` colour RAM to green (chip found),
red (not found), or yellow (info) per row.

**Embedded SID tune.** `bin/Triangle_Intro.sid` loaded at assemble time
at `:37-39`, played from the raster IRQ when `sid_music_flag` is set,
toggled on/off by keypress. V1.1 main loop is silent.

**Restart without reset.** `do_restart:` at `:1163` → `start:` at `:150`
performs a clean in-program restart from SPACE. V1.1 did have a
SPACE-to-restart via the raster IRQ (`SPACEBARPROMPT` at v1_1:409)
but it was minimal — no state reset of ZP variables, no debouncing
logic.

**Retry indicator (V1.3.83).** `print_retry_star:` at `:6866` appends
`*` after "6581 FOUND" / "8580 FOUND" when `checkrealsid` needed
more than one attempt — VIC bad-line DMA occasionally steals cycles
during D41B reads.

## 6. Bug Fixes of Note

Rather than a per-version dump (all 56 commits are in `git log`),
here are the substantive detection-logic fixes (drawn from STORY.md,
TODO.md, and inline source comments):

- **SIDFX early-return in `sidstereostart` (V1.3.59).** Without this,
  when SIDFX's SID2 happened to be PDsid (which doesn't answer the PD
  echo in `sidstereostart`'s scan), `sidstereostart` added it to
  `sid_list` at *every* mirror address in the D5xx range (8 entries),
  overflowing the 8-slot `sid_list` and corrupting slot 1's high byte.
  Fix: `cmp #$30; beq rts` at the top of `sidstereostart` (`:4848`).
- **`checkrealsid` retry loop (V1.3.83).** VIC bad-line DMA steals
  cycles during D41B reads, occasionally making a real 6581 fail
  the first attempt. 3-retry wrapper + retry star on screen.
- **Real-SID pre-check at Step 0.25 (V1.2.32).** Real 6581 LFSR at
  noise+$FFFF passes SwinSID Nano's cnt=7 test, producing false
  "SwinSID Nano" on every real 6581. Fix: run `checkrealsid` first;
  if confirmed, skip SwinSID Nano entirely.
- **Probe ordering in SIDFX SID1=UNKN chain (V1.3.68).**
  `sfx_probe_skpico` leaves `base+$1F = $FF` and `base+$1E = $E0`
  without cleanup on miss. These are exactly FPGASID's magic-cookie
  registers. Running skpico first corrupted FPGASID's identify mode.
  Fix: `checkfpgasid` must run before `sfx_probe_skpico`.
- **uSID64 two-read stability (V1.2.27).** A floating NOSID bus after
  the `$FF` write will read into `$E0-$FE` but drift; a real uSID64
  holds steady. Two-read tolerance `≤ $02` added.
- **FPGASID stereo scan (`fiktivloop` for `sidtype=$06/$07`).** V1.1
  called `fiktivloop` (noise-mirror test) for FPGASID too, which is
  the wrong method — FPGASID doesn't answer the noise-mirror. The
  current source removes the V1.1 `fiktivloop` branch in
  `s_s_lfpgasid`/`s_s_lfpgasid_2` and instead lets the outer loop
  continue scanning with `checkfpgasid` itself (see TODO.md).
- **FC3 cartridge false-positive C128.** V1.1 `check128` (v1_1:1832)
  only read `$D030` and dispatched `$FC` → C128 or `$FF` → C64.
  The Final Cartridge III drives `$D030` non-`$FF`, misdetecting as
  C128/TC64. Current `check128` (`:7444`) falls through to `$D0FE`
  on the non-`$FF` path and uses that as the final arbiter — FC3's
  `$D0FE` stays open-bus `$FF` so the fallback corrects to C64.
- **OPL `/IRQ` storm (V1.4.09).** Early OPL detection started Timer 1
  with IRQ unmasked; the OPL asserted `/IRQ` on cartridge bus, hung
  the KERNAL handler (which only ACKs CIA1). Fix: start T1 with
  MASK_T1 set (`reg $04 = $41`), clear with RST on exit.
- **OPL status heuristic (V1.4.19).** `$DF60` status must have upper
  bits clear (`& $E0 == 0`). Real OPL drives into the low-value
  range; bus noise almost always has at least one of bits 5-7 set.
- **MixSID C06 (6581@CS1 + ARMSID@CS2) — single CS2-DIS window
  (V1.3.74).** ARMSID snoops all bus writes regardless of chip
  select; DIS to CS1 would cause ARMSID to drive D41B with `$4E`,
  masking the 6581 OSC3. Fix: enter DIS via CS2 addresses
  (D43F/D43E/D43D), run `checkrealsid` inside that window, then
  exit by reading D43B (ACK).
- **D400+D500 mixed Swinsid/real-SID (V1.3.79).** Fixed via
  `s_s_arm_call_real` branching to `sfx_probe_dis_echo` when the
  primary is real SID ($01/$02) — SwinSID U at D500 echoes `'S'` at
  D51B and is correctly typed `$04` instead of being misidentified
  as 6581/8580.
- **"????" display bug for secondary ARMSID (V1.3.62).** When ARMSID
  was found in a SIDFX secondary, the display code called
  `print_sid_type_4` which reads `armsid_sid_type_h` — unpopulated
  for secondaries. Fix: skip the call when `data4 == $30` (SIDFX
  primary).

## 7. Memory Map / Zero-Page Usage

### Zero-page

V1.1 and current use the same ZP range (`$A2-$AF` + `$F6-$FF`) with
nearly identical names. Additions in the current source:

| ZP | Current                                                  | V1.1 |
|----|----------------------------------------------------------|------|
| `$B0` | `sid_music_flag` — IRQ-driven SID music on/off        | —    |
| `$B1` | `colwash_flag` — suppresses COLWASH on sub-screens     | —    |
| `$B2` | `retry_zp` — `checkrealsid` retry count                | —    |

`$A2-$AF` and `$F6-$FF` are otherwise unchanged in purpose; the
current file just documents them more thoroughly (see comments at
`:98-137`). `res_zp` (`$FF`) has taken on a tentative-SwinSID-Nano
role (`csn_not_usid64:` `:334`).

### Result table layout

V1.1 placed its result tables at a fixed `*= $1D00` (v1_1:2324).
Layout: `num_sids` (8 bytes), `sid_list_l/h/t` (8 bytes each),
`sid_map` (96 bytes, one per $20-step candidate slot).

Current source moved them to `*= $6000` (`:8197`). Layout is the
same `num_sids/sid_list_l/h/t/sid_map`, but with a **new 23-byte
`uci_resp` buffer** inserted between `sid_list_t` and `sid_map`
(`:8207`) to hold the U64 UCI GET_HWINFO response.

The move to `$6000` was forced by code growth: by V1.3.67 the code
segment reached `$55FF`, exactly 1 byte before the `sid_list` table
at `$5600` (an earlier relocation from `$1D00`). Relocating the data
table to `$6000` created 1536 bytes of headroom — see STORY.md §19
"Code Size Budget and Data Table Relocation".

### Code origin

V1.1 starts program code at `* = $080D` (v1_1:93). Current starts at
`* = $2400` (`:142`) — BASIC stub says `SYS 9216`. This move was
necessary because the embedded SID tune at `$1800-$2387` had to be
safely below the code, and there was no room below `$1800`.

### Embedded data blocks (new in current)

| Addr     | Contents                                                 |
|----------|----------------------------------------------------------|
| `$1800-$2387` | `Triangle_Intro.sid` (embedded SID tune, via `LoadBinary`) |
| `$0A00-...`   | TLR's `sid-detect2.prg` (embedded via `LoadBinary`)        |
| `$2400+`      | Main code                                                  |
| `$6000+`      | Result tables (`num_sids`, `sid_list_*`, `sid_map`, `uci_resp`) |

## 8. What V1.1 Had That's Gone / Consolidated

- **`checkswinmicro` disabled.** V1.1's `swinmicro:` jump table entry
  (v1_1:319) already had `jmp nosound` before the JSR, and the
  `checkswinmicro` routine itself (v1_1:1861) is effectively dead code.
  `checkswinmicro` still exists in the current source at `:7465`
  (verbatim port) but is not invoked from the main chain; the `$0A`
  dispatch code it used has been re-allocated to BackSID.
- **`SWINSID MICRO FOUND` string.** Declared in V1.1 as `swinsidmicrof`
  (v1_1:2237). Still declared in the current source at `:7917` but
  unreferenced.
- **`checksecondFPGA` routine.** V1.1 (v1_1:1550) had a second-FPGASID
  scanner that was never usefully called (see the V1.1 `s_s_lfpgasid`
  block at v1_1:1255 — the `jsr checksecondFPGA` is commented out).
  Ported verbatim to the current source at `:5836` but unreferenced.
  Superseded by the outer-loop fiktivloop + `checkfpgasid` redispatch.
- **"TODO: check for FCiii, and skip like 128 check!!"** (v1_1:19) —
  implemented. Current `check128` handles FC3 correctly (see §6).
- **"TODO: SFX Sound expander - x-soundfx.asm"** (v1_1:23) —
  implemented as `checkfmyam` (§3).
- **"TODO: ARM2SID detection - x-arm2sid.asm"** (v1_1:24) —
  implemented (§3, §5 via `armsid_get_version` + info page).
- **"TODO: ULTISID (U64) - asked"** (v1_1:29) — implemented via UCI
  (§4).
- **"TODO: Swinsid Nano - lav high/low"** (v1_1:34) — implemented via
  2-stage OSC3 activity profile (§3).
- **"TODO: MISTer C64 check", "MIDI test", "DTV Test", "128 DCR test"**
  (v1_1:25, 31-33) — still not done. Out of scope; MISTer and DTV are
  hard-to-acquire hardware, MIDI/128 DCR are niche.
- **Stereo `fiktivloop` FPGA paths.** V1.1 had `s_s_lfpgasid`/
  `s_s_lfpgasid_2` blocks (v1_1:1254-1269) that jumped into
  `fiktivloop` for FPGASID. Current source does not — FPGASID scan is
  now inline via the `s_s_l1` outer loop which redispatches through
  `checkfpgasid`.
- **V1.1 stereo colour/row printing** was a simple 8-entry loop in
  `sidstereo_print`. Current has a much wider dispatch in the same
  routine (`:5932`) covering all the new `sid_list_t` codes
  (`$09` PD, `$0A` BackSID, `$0B/$0E` SIDKick-pico, `$0C` KungFuSID,
  `$0D` uSID64) plus per-chip extras (ARM2SID slot map display,
  SKpico FM annotation, FPGASID CPLD/FPGA revision).
